Block Replace

This plug-in enables fast replacement of blocks.

Getting Started|Install/Uninstall|Mappings.xml File Details|Contact|Version History

Getting Started

To launch the plug-in, use the following command(s)

UpdateAttributes
Prompts the user for a drawing frame.

There is also a lisp version available for automated purposes: "uatts"
It comprises the following in a .lsp file:
(defun c:uatts (/ ss)
(setq ss (ssget "_X" (list (cons 0 "INSERT")'(66 . 1))))
(updateatts ss)
);uatts
SortRevisions

Prompts the user for three things:

  1. A Release Note (RN)
  2. A Release Date
  3. A drawing frame object

There is also a lisp version available for automated purposes: "srevs". It comprises the following in a .lsp file:



(defun c:srevs (/ ss rn rd)
(setq ss (ssget "_X" (list (cons 0 "INSERT")'(66 . 1))))
(setq rn "RN240129")
(setq rd "04-Mar-14")
(sortrevs ss rn rd)
);srevs

Usage

When the Block Replacement plugin has loaded you can easily replace one block with another and keep attributes the same providing the information is stored in the Mappings.xml file.

Installation and Uninstallation

Uses the Autoloader mechanism.

Mappings.xml File Details

This file was developed and released by Chillme1 on The Swamp and released at the following thread location: http://www.theswamp.org/index.php?topic=42890.0

It needs to be laid out like this:

        <mapping>
          <oldblock name="A2 BORDER" newname="Replacement_A2_Block" path="C:\The\path\to\Replacement_A2_Block.dwg">
            <attributes>
              <attribute name="DRAWING NUMBER 1" newname="DRAWING NUMBER 1"/>
              <attribute name="DRAWING NUMBER 2" newname="DRAWING NUMBER 2"/>
              <attribute name="TITLE" newname="TITLE 2"/>
              <attribute name="SHEET NUMBER 1" newname="SHEET NUMBER 1"/>
              <attribute name="SHEET NUMBER 2" newname="SHEET NUMBER 2"/>
              <attribute name="NUMBER OF SHEETS 1" newname="NUMBER OF SHEETS 1"/>
              <attribute name="NUMBER OF SHEETS 2" newname="NUMBER OF SHEETS 2"/>
              <attribute name="CERTIFIED" newname="CERTIFIED"/>
              <attribute name="CHECKED" newname="CHECKED"/>
              <attribute name="DRAWN" newname="DRAWN"/>
              <attribute name="MATERIAL" newname="MATERIAL"/>
              <attribute name="TOLERANCE" newname="TOLERANCE"/>
              <attribute name="SCALE" newname="SCALE"/>
              <attribute name="PROTECTIVE FINISH" newname="PROTECTIVE FINISH"/>
              <attribute name="DIMS IN" newname="DIMS IN"/>
              <attribute name="SURFACE TEXTURE" newname="SURFACE TEXTURE"/>

I have found the easiest way of creating this file is to use an Excel file after having opened the output file from the "ATTOUT" Express Tools command; an example of which you can find in the resources directory

NOTE: It is technically possible to add the ability to map this information to this file using the .NET framework.

Check the myCommands.vb file in the source folder for a method called "GetPointsFromUser" which shows how we do this to add a new block the BlockReplace.xml file.

Known Issues

For reasons unknown, Scriptpro will report the script has run successfully, even though it actually didn't.

The Drawing being processed can only have one block with attributes in. Any others will cause the code to error.
